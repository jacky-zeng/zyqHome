# 项目代码目录规范

本次开发严格遵循固定目录拆分规则，前后端代码完全分离，禁止目录混用，具体目录分工如下：

- **backProject 文件夹**：专属Go语言后端代码目录，仅承担接口开发职责，不包含任何前端页面相关代码，所有后端逻辑仅对外提供标准化接口，不处理前端渲染逻辑。

- **webVue 文件夹**：专属Vue3前端代码目录，负责页面渲染、交互逻辑实现、静态资源管理，通过调用backProject提供的后端接口获取数据

# 核心开发准则

**核心原则：前后端彻底解耦，模块拆分清晰，职责边界明确**

1. **代码模块切分**：编写代码时严格按业务功能、技术职责拆分独立模块，避免代码冗余、耦合，后端按接口业务域拆分，前端按页面组件、工具函数、路由模块拆分，提升代码可读性与后期维护效率。

2. **资源分离要求**：严格分离业务代码与静态资源，前端静态资源（图片、样式、插件等）统一归类存放，不与业务逻辑代码混杂；后端仅保留接口逻辑代码，无任何前端相关资源文件。

3. **前后端交互规则**：后端专注接口开发，保证接口规范、参数清晰、返回数据格式统一；前端仅负责调用对应接口，接收数据后完成页面渲染与交互反馈，全程不跨目录编写代码，不突破前后端职责边界。

# 代码编写要求

所有代码需符合行业通用规范，Go后端代码遵循Go语言官方编码规范，接口设计遵循RESTful风格；Vue3代码遵循组合式API规范，组件化开发，代码注释简洁到位，关键逻辑添加说明，方便后续迭代与协作。
